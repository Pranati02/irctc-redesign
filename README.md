# IRCTC Website Redesign 🚂

A clean, modern redesign of the IRCTC (Indian Railway Catering and Tourism Corporation) booking portal — built as a frontend portfolio project.

## 🎯 Goal
The original IRCTC website is notoriously cluttered and confusing. This redesign keeps the **same blue and orange brand colors** while drastically improving usability, layout, and visual hierarchy.

## ✨ What's Improved

| Original IRCTC | This Redesign |
|---|---|
| Cluttered navbar with 20+ links | Clean sticky nav with 5 key links |
| Tiny, hard-to-read form fields | Large, clearly labeled inputs |
| No visual grouping | Card-based layout with clear sections |
| Confusing tab layout | Clear tab bar: Book / PNR / Trains / Cancel |
| No autocomplete for stations | Live station autocomplete (24 major stations) |
| Hard to find popular routes | Dedicated "Popular Routes" section |
| No search feedback | Toast notifications + loading state |
| Not mobile-friendly | Fully responsive across all screen sizes |

## 🛠️ Tech Stack
- Pure HTML5, CSS3, Vanilla JavaScript
- Google Fonts: Plus Jakarta Sans + Noto Sans
- No frameworks, no dependencies — works offline

Just open `index.html` in any browser. No build step needed.

> Deploy to GitHub Pages: Settings → Pages → Deploy from `main` branch → `/root`

## 📸 Features
- **Sticky navbar** with IRCTC branding
- **Hero section** with animated track pattern
- **Booking card** with 4 tabs: Book / PNR Status / Train Search / Cancel
- **Station autocomplete** with 24 major Indian cities
- **Swap button** to reverse origin/destination
- **Popular Routes** with live availability indicators
- **Quick Services** grid (e-Catering, Retiring Rooms, etc.)
- **Scrolling announcements ticker**
- **Stats section** (1Cr+ daily tickets, 13,000+ trains)
- **Responsive footer** with helpline
- **Toast notifications** for user feedback
- Fully **responsive** — works on mobile, tablet, desktop

## 🎨 Design Decisions
- **Colors preserved**: IRCTC Blue `#003580` + Orange `#FF6600`
- **Typography**: Plus Jakarta Sans for headings, Noto Sans for body
- **Whitespace**: Generous padding to reduce cognitive load
- **Focus states**: Clear blue glow on all interactive fields
- **Hover states**: Subtle lift + shadow on cards
